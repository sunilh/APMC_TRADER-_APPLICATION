Step-by-Step Fix for Replit "Port not exposed" Error
✅ 1. Make sure you're binding to process.env.PORT — only
Don't fallback to port 5000 at all — Replit will break if the wrong port is used.

Update your line like this:

ts
Copy
Edit
const port = Number(process.env.PORT);
No default. Replit always provides process.env.PORT, so it's safe.

✅ 2. Ensure you're binding to 0.0.0.0 (you already do this ✅)
You're already doing this:

ts
Copy
Edit
httpServer.listen(port, '0.0.0.0', () => {
👍 That part is correct.

✅ 3. Add a simple / route before anything else
Replit's health check hits / — so add this before anything async (like Vite):

ts
Copy
Edit
app.get('/', (_req, res) => {
  res.send('🟢 Server is running.');
});
✅ 4. Confirm .replit file has correct run command
In .replit, you should have:

toml

run = "npx tsx index.ts"
Or, if you're compiling with tsc:

toml

You can check by clicking the “⋮” next to the Run button > Show hidden files > .replit.

✅ 5. Avoid Awaiting setupVite() If It Blocks Server Startup
If setupVite() sets up the dev server and holds the event loop (common with Vite's middleware), Replit might never detect the port bind.

👉 Temporarily comment out Vite and try a plain Express setup:

ts
Copy
Edit
// Comment out for testing
// await setupVite(app, httpServer);
Then test again. If it works now, the problem is Vite never returning in time — in that case, use a static build for Replit.

✅ 6. Final Minimal Server Code for Replit
Here’s a version that always works on Replit:

ts
Copy
Edit
import express from 'express';

const app = express();
const port = Number(process.env.PORT);

app.get('/', (_req, res) => {
  res.send('🟢 Hello from Replit!');
});

app.listen(port, '0.0.0.0', () => {
  console.log(`✅ Listening on http://localhost:${port}`);
});
Test this first. If it works, gradually reintroduce your routes and Vite config.
